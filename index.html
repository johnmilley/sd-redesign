AI-Assisted Development in the SD Program

Integrating AI coding tools into the Software Development (Co-op) program at CNA.
Thesis Curriculum Tools Textbooks Pedagogy Community Next Steps

Thesis

AI coding tools should be adopted into the SD program. First year already teaches the fundamentals needed to use them. Second and third year should lean into them heavily. The result is better graduates and a stronger tech industry in NL.

This is already happening elsewhere. CMU launched a course where students build full applications without writing code manually. Anthropic partnered with CodePath to put Claude Code at the centre of CS programs reaching 20,000+ students, including HBCUs. Stanford is teaching "vibe coding" as continuing education.

In Canada, the conversation is moving too. Policy Options argues the country needs a national strategy for generative AI in higher education. Right now there isn't one—it's a patchwork. The NL government is already investing nearly $2M in AI micro-credential training for local businesses. The demand is here. The program should match it.

Years 2 & 3 — Integration proposed

With foundations in place, upper years should use AI tools as a standard part of the workflow. Not instead of manual work—alongside it. Always instructor-facilitated. Manual skills still tested.

Tools

Several options exist. Claude Code is the recommendation because it runs at the command line (reinforces Year 1 skills) and supports educational licensing.

Claude Code recommended
CLI-native. Terminal-first workflow, git integration. Reinforces fundamentals.
GitHub Copilot
IDE-integrated autocomplete. Widely adopted in industry.
Cursor
AI-first editor. VS Code fork with built-in chat and editing.
Windsurf
AI IDE with agentic workflows.

Students should know the alternatives. The field moves fast and tools change. What matters is the skill—working effectively with AI—not any single product.

Student-Built Textbooks

Students use AI tools to build their own course materials. They learn the tool by building something they actually need.

This does two things: teaches the technology and builds the habit of creating your own learning resources. In this field, the docs you need don't always exist. Building them yourself is a career-long skill.

Software development changes constantly. Students who can teach themselves—who can take unfamiliar material and build working knowledge from it with AI assistance—are the ones who stay current. That's the lifelong learning piece. Not a buzzword. A practical skill we can actually teach.

Pedagogy

This doesn't lower the bar. It raises the ceiling.

What stays the same

  • Manual coding is still assessed—students write code without AI
  • Theory and concepts tested independently
  • Instructor facilitation in all AI-assisted work

What changes

  • Prompting and specification writing become core skills
  • Code review gets real—evaluating AI output for correctness and security
  • Project scope increases to match real-world complexity

Assessment covers three things: does the student understand the concept, can they do it manually, and can they use AI to build at a higher level. All three, every time.

Community & Industry

Co-op work terms should have a local focus where possible. Students using these tools on real projects in NL demonstrate the technology directly to employers and stakeholders.

  • Graduates who can work with AI tools are immediately more useful to local employers
  • Demonstrating the tech to local shareholders generates buy-in and knowledge building
  • ACOA's Regional AI Initiative is already funding Atlantic Canadian businesses adopting AI—our graduates should be ready to work in that environment
  • Early adoption positions the SD program as industry-aligned
