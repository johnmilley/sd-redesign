AI-Assisted Development in the SD Program

Integrating AI coding tools into the Software Development (Co-op) program at CNA.
Thesis Curriculum Tools Textbooks Pedagogy Community Next Steps

Thesis

AI coding tools should be adopted into the SD program. First year already teaches the fundamentals needed to use them. Second and third year should lean into them heavily. The result is better graduates and a stronger tech industry in NL.

This is already happening elsewhere. CMU launched a course where students build full applications without writing code manually. Anthropic partnered with CodePath to put Claude Code at the centre of CS programs at 20,000+ student institutions, including HBCUs. Stanford is teaching "vibe coding" as continuing education.

These aren't experiments anymore. They're curriculum.

Years 2 & 3 — Integration proposed

With foundations in place, upper years should use AI tools as a standard part of the workflow. Not instead of manual work—alongside it. Always instructor-facilitated. Manual skills still tested.

Student-Built Textbooks

One concrete use case: students build their own course materials using AI tools. This isn't hypothetical. Google's LearnLM team is already researching AI-personalized textbooks. UCLA is piloting AI-generated course texts.

For our program, the approach is more hands-on. Students use Claude Code (or similar) to generate, edit, and maintain reference materials for their own courses. They learn the tool by building something they actually need.

This does two things at once: it teaches the technology, and it builds the habit of creating your own learning resources. In this field, the documentation you need doesn't always exist. The ability to build it yourself is a career-long skill.

Software development changes constantly. Frameworks, languages, and best practices shift every few years. Students who can teach themselves—who can take unfamiliar material and build working knowledge from it with AI assistance—are the ones who stay current. That's the lifelong learning piece. Not as a buzzword, but as a practical skill we can actually teach.

Pedagogy

This doesn't lower the bar. It raises the ceiling.

What stays the same

  • Manual coding is still assessed—students write code without AI
  • Theory and concepts tested independently
  • Instructor facilitation in all AI-assisted work

What changes

  • Prompting and specification writing become core skills
  • Code review gets real—evaluating AI output for correctness and security
  • Project scope increases to match real-world complexity

Assessment covers three things: does the student understand the concept, can they do it manually, and can they use AI to build at a higher level. All three, every time.

Community & Industry

Co-op work terms should have a local focus where possible. Students using these tools on real projects in NL demonstrate the technology directly to employers and stakeholders. That builds connections and awareness.

  • Graduates who can work with AI tools are immediately more useful to local employers
  • Demonstrating the tech to local shareholders generates buy-in and knowledge building
  • Early adoption positions the SD program as industry-aligned

Next Steps

  1. Secure educational licensing (starting with Claude Code)
  2. Pilot in one or two upper-year courses
  3. Develop assessment rubrics for AI-assisted work
  4. Gather feedback from students and co-op employers
  5. Expand based on results
